Notebook CSS

Notebook is a lightweight CSS boilerplate that's dead simple to use.

Using Vanilla CSS only, Notebook requires no dependencies, no preprocessers, and no headaches.

A Few Examples

Notebook provides styles for common elements in a minimal, neutral way which makes it incredibly easy to edit for future uses.

Typography Scale

The type is set up to scale by setting a variable for --default-size and --text-scale. Once these are set, the headings and body type will be mathematically scaled to match.

Variables are currently available for the following:

Variable Value
--default-size 1rem
--text-scale 1.2
--default-line-height 1.4
--text-xs calc(var(--default-size) / var(--text-scale))
--text-sm var(--default-size)
--text-md calc(var(--text-sm) * var(--text-scale))
--text-lg calc(var(--text-md) * var(text-scale))
--text-xl calc(var(--text-lg) * var(--text-scale))
--text-xxl calc(var(--text-xl) * var(--text-scale))
--text-xxxl calc(var(--text-xxl) * var(--text-scale))

For more information or experimentation with modular type scale, visit modularscale.com

Feel free to edit --font-primary, --font-secondary, --font-monospace, and --font-color as needed.

Layouts and Containers

Breakpoints and containers, like Notebook's typography, are based on an initial value and then scale accordingly.

Variable Value
--container-mobile 480px
--container-tablet calc(var(--container-mobile) * 1.5)
--container-desktop calc(var(--container-tablet) * 1.5)

Breakpoints are set to correspond with container sizes. Currently, @media attributes don't work with native CSS variables as they do in Sass or other preprocessers. So Notebook is designed to scale with the content.
